## Descripción del Proyecto
Este repositorio contiene la práctica individual correspondiente al Taller 4. El objetivo principal es aplicar herramientas de gestión de proyectos para la investigación reproducible, practicar la colaboración y el control de versiones mediante Git y GitHub, y documentar flujos de trabajo computacionales.

## Fases del Taller
El proyecto documenta el aprendizaje en cuatro áreas clave:
1. **Gestión de Proyectos (Quire):** Planificación y estructuración de tareas para el taller.
2. **Control de Versiones Avanzado (Git/GitHub):**
   - Creación de ramas y fusiones (*branching & merging*).
   - Simulación y resolución de conflictos de fusión (Merge Conflicts).
   - Uso de `git commit --amend` para corrección de errores en el historial.
   - Recuperación de archivos eliminados mediante la navegación en el historial (`HEAD~1`).
3. **Flujos de Trabajo en Mermaid:** Creación de un `diagram.mermaid` que ilustra un pipeline reproducible basado en el protocolo TIER (Simulación ➡️ Limpieza ➡️ Análisis ➡️ Visualización).
4. **Flujos de Trabajo en Draw.io:** Ampliación del diagrama base incorporando una etapa de "Validación Automatizada de Datos" (Programación Defensiva), inspirada en las metodologías de *Kitzes et al. (2017)*.

## Estructura del Repositorio
- `/analisis/` - Scripts de modelado econométrico y estadística descriptiva.
- `/analisis_exploratorio/` - Scripts de exploración y parametrización visual.
- `/src/` - Funciones modulares de utilidad.
- `diagram.mermaid` - Código fuente del diagrama de flujo de trabajo.
